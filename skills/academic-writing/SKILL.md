# 水利/水电学术写作技能

## 这个技能是做什么的

把水利工程、水电工程领域正式书面成果的**结构范式**和**行文风格**固化下来,让 Claude
在写论文、申请书时,产出的初稿在章节骨架、语言风格、图表符号、量化论证、引用著录上
直接符合本领域的惯例,而不是通用的"看起来对但不地道"的写法。

本技能不替代作者的专业判断——数据、机理解释、创新点必须来自作者。技能负责的是:把
作者的素材组织成符合规范的成品,并在结构和文风上保持一致、可复核。

## 使用前必读:先确认三件事

动笔前先明确下面三点。如果用户已经说清楚,直接开始;如果没有,用一句话确认再写,不要
凭空假设。

1. **体裁**——是学位论文、期刊论文,还是项目申请书?三者结构差异很大(见下)。
2. **学科侧重**——偏水利(河流动力学、泥沙、水文、河床演变、防洪、水资源)还是偏
   水电(水工结构、大坝、机组、调度、发电、泄洪消能)?术语和论证重点不同,见
   `references/domain-terms.md`。
3. **手头素材**——是从零起草,还是基于作者已有的数据/图表/初稿续写或修改?基于素材
   时,严禁编造数据、文献或结论;缺什么就明确标注 `【待补:...】`,交回作者填。

## 三种体裁的选择与骨架

根据体裁读取对应的详细模板文件,不要把三种混在一起套用:

| 体裁 | 何时用 | 详细模板 |
|------|--------|----------|
| 学位论文(硕士/博士) | 毕业论文、开题报告扩展、学位申请 | `references/genre-dissertation.md` |
| 期刊论文 | 投稿中英文期刊、会议论文 | `references/genre-journal.md` |
| 项目申请书(国基金等) | 立项依据、研究方案、面上/重点/青年项目 | `references/genre-grant.md` |

**学位论文骨架(概要):** 封面要素 → 中文摘要 + ABSTRACT(各附关键词/Keywords)→
目录 / 附图附表清单 / 主要符号对照表 → 第一章 绪论(问题的提出 → 国内外研究进展[分
主题综述]→ 尚存在的问题 → 研究内容 → 技术路线)→ 方法/模块章 → 若干结果章(每章末
"本章小结")→ 结论与展望(主要结论 + 研究展望)→ 参考文献 / 致谢 / 附录。学位论文有
**两种研究范式**:实验/机理类(方法章 + 结果章,现象→机理→公式)与建模/优化类(研究
主线贯穿的具名模块章,含研究区概况与应用实例)。二者正文骨架不同,务必先判断属于哪类
再展开,详见 `references/genre-dissertation.md`。

**期刊论文骨架(概要):** 题目(中英)→ 作者与单位 → 摘要 + Abstract → 关键词 →
引言 → 材料与方法(或实验/模型)→ 结果与分析 → 讨论 → 结论 → 参考文献。

**申请书骨架(概要,以国基金正文为例):**(一)立项依据与研究内容[1 立项依据(研究
意义 / 关键科学问题的提出 / 国内外研究现状及发展动态分析 / 现有研究不足与切入点)→
2 研究内容、研究目标、拟解决的关键科学问题 → 3 研究方案及可行性分析(研究方法 / 技术
路线 / 可行性分析)→ 4 特色与创新之处 → 5 年度研究计划及预期研究结果];(二)研究
基础与工作条件。

## 核心行文规范(三种体裁通用)

下面是本领域正式成果最鲜明的写法特征,任何体裁都要遵守。更细的规则和正反例见
`references/style-conventions.md`,写作时务必参考。

### 1. 中英文双语要素
- 学位论文与期刊论文必须**中英文摘要对照**,并各附中文关键词与英文 Keywords。
- 图、表题注**中英文双语**并列:中文题注在上、英文在下。例:
  `图 2.17 倾斜拍摄示意图(单桩)` / `Fig. 2.17 Schematic diagram of the tilted shooting (monopile)`
- 专业术语**首次出现**时给出中文全称、英文全称及缩写,此后统一用缩写。例:
  高桩承台(High-Rise Structure Foundation, HRSF)、紊动能(turbulent kinetic
  energy, TKE)、雷诺时均(RANS)、大涡模拟(LES)。

### 2. 摘要与结论的"分条量化"范式
摘要按"背景与问题 → 研究方法(采用……相结合的方法)→ 主要研究结论如下: → 分条编号
结论"组织。每条结论以**动词短语概括**领起(揭示了 / 研究了 / 探明了 / 建立了 / 提出了
/ 对比了 / 分析了),随后给出**具体量化结果**支撑,而非空泛描述。结论章、各章"本章小结"
同此范式。反面写法(禁止):"本文对冲刷问题进行了研究,取得了较好效果。"

### 3. 量化表达优先
凡能量化处一律给数据:百分比、倍数、区间、误差。例:"约为单桩的 60%~70%""预测误差
小于 20%""坡度从 30°变化到 18°""1.36~2.35 倍等效直径"。区间用波浪号 `~`,不用
连字符。

### 4. 图表编号与"先引后现"
图表编号采用"章号.序号"(图 2.17、表 3.1、式(3-2))。正文必须**先文字引用、再出现
图表**("如图 3.1 所示……"),不得图表先于引用出现。每张图表都要在正文里被解读,不能
只放图不说话。

### 5. 符号与单位规范
- 学位论文设"主要符号对照表",列出全部变量、含义、单位。
- 变量用斜体(如 *d*₅₀、*U*_c、*Re*),下标规范;矢量、张量按惯例。
- 单位用国际单位制并规范书写(m/s、cm、Pa、m³/s、kg/m³)。同一物理量全文单位一致。

### 6. 引用与文献著录
- 正文引用用上标数字、按出现顺序编号:`……普遍发生清水冲刷与河床粗化[2, 3]。`
- 参考文献按引用顺序列出,著录格式统一(期刊:作者. 题名[J]. 刊名, 年, 卷(期): 页码)。
- **绝不编造文献**。作者未提供的引用一律标 `【待补:文献】`,由作者补齐。

### 7. 论证性连接词
用逻辑连接词构建因果与递进链条:然而、因此、此外、由于、说明、导致、表明、结果表明、
综上。方法步骤用 (1)(2)(3) 编号。避免口语与主观夸张(如"非常好""很厉害")。

## 工作流程

1. **确认体裁与学科**(见上)。读取对应的 `genre-*.md` 与 `domain-terms.md`。
2. **基于素材、不凭空造**。有作者数据/初稿时,严格依据其内容组织;信息缺口用
   `【待补:...】` 标注,集中列在文末"待补清单"里,方便作者一次补全。
3. **先搭骨架,再填内容**。先输出完整章节目录(骨架)让作者确认,再逐节展开,避免
   一次性写偏。长文档分节交付。
4. **保持可复核**。凡涉及数据、图表、公式、文献处,清楚交代来源或标注待补,不把
   不确定的内容写成确定结论。这一条对学位论文与申请书都是硬要求。
5. **自查**。完稿后对照 `references/style-conventions.md` 末尾的检查清单逐项核对
   (双语要素、图表编号、量化表达、符号一致、引用规范、有无编造)。

## 交付格式

- 默认用 Markdown 输出,便于作者审阅和继续编辑;中英摘要、图表题注、公式编号都在
  Markdown 里体现。
- 若作者明确要求 Word 成品(如"给我一份 docx""要能直接交"),再转为 .docx,并套用
  相应体裁的排版惯例(此时调用 docx 相关技能)。**若是河海大学学位论文**,直接用
  hhu-thesis-docx 技能生成——它已按《河海大学学位论文编写格式规定》固化了字体、行距、
  页眉页脚、图表编号等全部排版规格,比通用 docx 更贴合。
- 初稿成形后,可用 hhu-thesis-review 技能对前言/摘要/问题提出/结论/参考文献五部分做
  分级审稿自查(河海博士论文)。
- 每次交付附一份"待补清单",列出所有 `【待补:...】` 项,让作者清楚还差什么。
