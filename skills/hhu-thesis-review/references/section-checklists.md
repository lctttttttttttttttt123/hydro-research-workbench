# 四部分审查清单（前言 / 摘要 / 问题的提出 / 结论）

参考文献单独成篇，见 `gb-t-7714.md`。

每条检查项标注了默认严重度，但要结合上下文判断——同一问题在不同论文里轻重可能不同。给意见时按 SKILL.md 的"逐条意见"格式：定位 → 为什么 → 怎么改。

---

## 总则：先做"读者代入式判断"，再走条目清单（最重要）

下面每一块都列了具体检查项，但**不要把审稿做成机械的要素打勾**。要素清单只能查出"缺了什么"，查不出"写得到不到位"——比如创新点四要素齐全、却依然让人读不出新意，清单会放行，但它其实不合格。

所以审每一块的**第一步**，是先放下清单，**代入一个真实读者的视角，把这一块从头读一遍，然后老实回答"我读完到底懂没懂、感觉到没感觉到"**。这件事 AI 本来就会做——你具备这个判断力，只是默认不会主动做，这里要求你**显式地做出来、并把判断结论写进意见**。

代入的读者身份：**一位熟悉本领域、但完全没读过这篇论文的同行评审**。带着这个身份读，对每一块问下面的核心问题（每块的具体问法见各自清单的第 0 条）：

- **前言**：读完背景，我能不能顺着它自然走到"所以才必须做这件事"？读完创新点，我能不能一句话说清每条"新"在哪？
- **摘要**：读完，我能不能抓住这篇到底"做了什么、得到了什么具体结果"，而不只是"研究了某个问题"？
- **问题的提出**：这一节读完，我手里有没有一个清楚的"待解决的科学问题"，还是只读到了一堆背景和意义？
- **结论**：每条结论是不是一个"凝练的判断"，还是又把方法过程复述了一遍？

**判断的关键动作**：如果读完"说不清/没感觉到/抓不住"，**不要含糊带过，要把它当成一个问题报出来**，并定位到是哪一句、哪一条让你卡住，再分析卡在哪（是写成了内容复述？动词没承载意思？还是真有东西但被埋没了？）。**"我作为同行读完看不出新意"本身就是一条有效的、必须写出来的审稿意见**——这恰恰是机械清单审不出、而最值得指出的问题。

做完这步读者代入判断、把结论记下来之后，再走下面每块的具体条目清单做补充检查。两者结合：读者代入抓"到不到位"，条目清单抓"缺不缺项"。

## 一、前言（foreword）

前言的定位是"为什么做、新在哪"，不是把摘要的结论提前抄一遍。

**第 0 条 · 读者代入判断（先做，最重要）**

带着"同行评审"身份通读前言，尤其是创新点，逐条做下面四项判断，把结论显式写出来：

1. **新意可辨性**：这一条创新点读完，你能不能用一句话说清它到底"新"在哪——是新机理、新方法、新对象，还是新数据？
   - 如果**说不清**，要指出是哪一条、并判断它属于哪种病：
     - **写成了"研究内容的复述"**：通篇是"研究了/分析了/理清了 X"，只说做了什么研究，没说贡献了什么新认知。这是最常见的病——把"我做了什么"误当成"我创新了什么"。读者读完只知道你研究了某问题，感觉不到这是"别人没做到、你做到了"的东西。
     - **真有新意但被埋没**：其实有突破，但被淹在平淡表述里，或被一句话带过。
2. **动词检验**：这一条的领头动词（如"理清/分析/构建/发展"）是在描述**动作**，还是承载**突破**？纯中性动作动词常常让新意读不出来。若是，提示换用能点明贡献性质的动词——例如新意在机理就用"揭示了/阐明了"，在方法就用"突破了/解决了……局限"，让动词本身把"新"扛出来。
3. **被写薄的核心**：有没有哪一条其实是最硬的创新（典型信号是出现"解决了某难题""突破了某瓶颈"），却被一句话轻轻带过、没有任何结果或对比撑起来？把它点出来——这往往是全篇最该浓墨重彩、却被作者自己写薄了的真正创新核心，提示用具体结果（降了多少误差、比谁好、好在哪）把它立起来。
4. **逻辑层次**：几条创新点之间有没有可辨的结构（递进或分工，如"机理发现 → 定量归因 → 方法工具"）？如果它们其实隐含这个结构、却平铺成几个并列任务、没点破，提示把递进关系理顺，让读者看出这是一个有组织的贡献体系，而非几件并列的事。

**怎么给意见**：上面任何一项的判断结论，都要落成可定位的意见——指到具体哪一条、说清为什么（哪种病）、给出方向。注意方向只能提示"你需要点出 X"（如"这里需要点明你的模型比既有方法误差降低的幅度"），**不要替作者编造他研究里没有的具体内容或数字**——你不知道他的真实贡献，只能指出他需要补什么。

---

做完第 0 条，再走下面的条目清单做补充：

1. **漏斗式背景【建议】**：背景是否按"宏观需求 → 领域科学缺口 → 本文具体对象"逐级收窄。若直接从具体对象讲起、缺宏观或缺缺口环节，提示补齐。好的前言能让读者顺着背景自然走到"所以才需要做这件事"。

2. **创新点核心两要素【建议】**：逐条创新点检查是否讲清了**"做了什么"**和**"得到什么结果"**这两个核心。这两者缺一就标出（如只说做了实验却不说发现了什么）。这是第 0 条"新意可辨性"的底线版本——要素齐全只是合格的**必要**条件，不是充分条件；要素齐了但读者仍读不出新意（第 0 条判断），依然要按第 0 条报出来。
   - **软化说明**：传统上还期望创新点带"领域局限句"开头和"价值收束句"结尾（即"局限→做法→结果→价值"四段式），但这两项只作**加分项**，缺了不算缺陷；创新点的结构变体（比如先讲结果再补做法）也**不报为问题**。只要"做了什么+得到什么"清楚，就算合格。

3. **创新点标题信息量【可选】**：小标题若是完整陈述句（动词+宾语+结果，单看标题即知贡献）更好；若是空泛名词词组（如"关于X的研究"），可建议改写为能独立成立的贡献句。

4. **与摘要去重【硬伤/建议】**：把前言创新点与摘要结论逐条比对。若大段逐句雷同 → 配合红线第 1 条按【硬伤】处理；若只是部分重合 → 【建议】前言侧重"为什么做/新在哪"，把定量结论留给摘要。

5. **拔高词检测【建议】**：扫描"首次/填补空白/不可替代/深刻变革/严峻挑战"等自我拔高词。
   - **降级规则（避免误报）**：单个"首次"等若**有数据或具体证据支撑**，**不报**；只在两种情况下提示——(a) 该词**缺乏证据支撑**（空口拔高），或 (b) 同一部分内**密集出现三个以上**拔高词。报的时候列出位置，建议替换为可被证据支撑的中性表述。

## 二、摘要（abstract）

摘要的定位是"做了什么、得到什么结果"，核心价值在结论的硬度。

**第 0 条 · 读者代入判断（先做）**

带着"同行评审"身份读完整段摘要，问自己：**我读完，手里抓住的是几个具体的、带结果的结论，还是只知道这篇论文"研究了某个问题"？**

- 如果读完只留下"他研究了 X、分析了 Y、建立了模型"这类印象，**却说不出他到底得到了什么具体结果**，这就是摘要最常见的失败——把"研究内容"当成了"研究结论"。逐条指出哪些结论是空的（只有定性动词、没有可被记住的硬结果），这跟下面第 2 条的"结论数据化"是同一件事，但要从"读者记不记得住"的角度去判断，而不只是"有没有数字"。
- 同时判断：摘要里"最值得记住的那个结果"是什么？如果你作为读者都挑不出一个，说明摘要没有亮点，提示作者把最硬的那条结论提到更显眼、更具体。

做完第 0 条，再走下面条目清单。

1. **四段结构【建议】**：是否齐备"背景（1 段收窄到对象+方法）→ 分条结论 → 价值句 → 关键词"。缺哪段提示补。

2. **结论数据化【硬伤/建议，分类判断】**：逐条结论检查，但**按结论类型区别要求**：
   - **实测/实验类结论**（测了什么、对比了什么、影响规律）：**要求有具体定量结果**（数值/区间/阈值/比例）。只有定性动词（"揭示了""分析了""研究了"）而无任何数字的实测类条目 → 【建议】补关键数字。
   - **方法构建类、机理/概念类结论**（提出了某模型、阐明了某机制、构建了某框架）：**不强求数字**。只检查它是否给出了**判断性结论**（这个方法解决了什么、这个机理是什么），而不是停留在**过程复述**（"建立了模型并进行了验证"这种只说做了不说结论的）。是过程复述 → 【建议】改写为结论判断。

3. **条目—章节对应【建议】**：结论条数应与正文主章对得上。有结论无对应章、或有重要章节无对应结论，提示。

4. **关键词覆盖【建议】**：数量适中（约 3–8 个）；是否覆盖论文核心对象与核心方法；是否与题名、正文主题词一致。漏掉核心词（例如做鱼类研究却没有"鱼类"）要点出。

5. **关键词分隔符一致性【可选】**：全角"；"与半角";"、中英文标点是否统一。

6. **句长可读性【可选】**：单句过长、从句套从句的句子标出，建议拆分。

## 三、问题的提出（绪论 1.1）

这一节名叫"问题的提出"，就应该真的提出问题，而不只是堆背景。

**第 0 条 · 读者代入判断（先做）**

带着"同行评审"身份读完这一节，问自己：**读到最后，我手里有没有一个（或几个）清楚的、待解决的科学问题？还是我只读到了一堆背景、意义、和别人做过什么？**

- 如果读完**找不到一个明确的科学问题**，只感觉"这一节在讲这个领域有多重要、前人做了哪些工作"，那这一节就名不副实——它叫"问题的提出"却没提出问题。指出这一点，并判断真正的"问题"是被作者**遗漏了**，还是**散落在别处**（常见于被挪到 1.2 综述末尾）。
- 如果**找到了**问题，再判断它**尖不尖锐**：是具体到能对应后文研究内容的真问题（"现有 X 方法在 Y 条件下误差大"），还是空泛套话（"认知不足""缺乏系统研究"）？空泛的要提示改成可证伪、可对应研究内容的具体问句。

做完第 0 条，再走下面条目清单。

1. **名实相符【硬伤/建议】**（红线相关，重点）：本节是否真的收口到"科学问题"，还是通篇只有背景与意义。若真正的"尚存在的问题"被散放到 1.2 综述末尾、而 1.1 名不副实 → 建议在 1.1 末尾用一两句**显式收口**出待解决的科学问题。名实严重不符按【硬伤】，轻微按【建议】。

2. **三级漏斗 + 落点【建议】**：背景是否按三级（如 世界大河→某大河→本文河段；或 大领域→子领域→本文对象）收窄，并清楚落到"本文研究对象"，交代选题独特性/正当性。

3. **关键问题尖锐度【建议】**：若已列出待突破问题，检查是否空泛（"认知不足""缺乏框架"这类）。建议改成可证伪、可与研究内容对应的具体问句。

4. **问题—内容—结论闭环【建议】**（红线相关）：1.1 提出的科学问题，应能与研究内容（通常 1.3）、主要结论一一对应。断链处标出。

5. **引用堆叠【可选】**：多区段引用连排（如一句话挂 [2-6][7-12]）、单篇文献贡献无法分辨的，标出并建议拆分或点明代表性文献。

6. **综述越界【建议】**：大段纯文献综述塞进 1.1，使本节臃肿、背景与综述混杂 → 建议把综述移入专门的综述节，1.1 只保留导向性引用。

## 四、结论（结论与展望）

结论要凝练判断，展望要具体。

1. **结论—章节可追溯【建议】**：每条主要结论能否对回某一章的贡献。空泛、无对应的条目标出。

2. **是结论而非复述【硬伤/建议】**：条目是在给"凝练判断"，还是混入了方法/过程复述（"采用XX方法对XX进行了研究"这种）。纯过程复述按【硬伤】（结论章不该这样写），半复述半结论按【建议】精简为结论句。

3. **对比性/层次性【可选】**：是否有跨结果的对比判断（如近场 vs 远场、A 主导 vs B 次要），还是平铺流水账。有层次更好。

4. **结论 vs 创新点去重【建议】**：若结论之外另列"创新点/主要创新成果"，比对是否在重复申报同一贡献。重复的建议合并或分工。

5. **背景模板复用【硬伤】**（红线第 1 条）：结论开头是否又抄了一遍绪论背景段。是 → 硬伤，建议删去、直奔结论。

6. **展望质量【建议】**：每条展望是否"具体不足 → 具体方向"成对出现。空泛展望（"有待进一步研究""值得深入探讨"）逐条标记，鼓励点名具体的方法局限、数据局限或待验证假设。

7. **整体升华【可选】**：是否缺一段把各章结论串起来的机理总结或总体图景。有则更完整。
