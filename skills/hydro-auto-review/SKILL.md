# 成稿后自动审查(只标不改)

对标 Claude Science 的"后台审查器":把能机器判定的硬伤自动查出来——没依据的东西、不可
追溯的图、对不上的登记、残留的待补与 AI 味。**只发现、不修改。**

## 定位:审查者,不是修改者
- 只输出"问题清单 + 定位 + 分级",**绝不自动改动任何文稿或数据**。
- 只查**机器可判定**的硬伤(可追溯性、一致性、待补残留、AI味残留、格式痕迹、复现完整性)。
- **学术判断**(创新点读不读得出、论证是否充分、机理解释对不对)不在此列——那仍交给
  作者和 hhu-thesis-review。两者互补:本技能查"硬伤",hhu-thesis-review 审"内容质量"。

## 工具(scripts/auto_review.py)
```bash
python3 scripts/auto_review.py --root <项目目录>
python3 scripts/auto_review.py --root <项目目录> --out logs/review_report.md   # 另存报告
```
发现"硬伤"退出码为 1(便于接入自动流程/定时任务)。

检查项:
- **[可追溯]** figures/ 里的图缺同名 `.meta.json`(来源不明,视为不可信)→ 硬伤。
- **[待补]** manifest.pending 未清零;drafts/outputs 残留 `【待补/待核/润色存疑】` → 硬伤。
- **[一致性]** manifest 登记的产出在磁盘找不到 → 硬伤。
- **[AI味]** drafts 残留高频中文学术 AI 腔 → 建议(转 hydro-deai-polish 顺一遍)。
- **[格式]** 表情符号(应移除)、成对花引号、疑似滥用的破折号堆叠 → 建议/提示。
- **[复现]** env/requirements.txt 未锁定 → 建议。

## 怎么用
1. 成稿后(或交付前)对项目工作区跑 `auto_review.py`。
2. 按报告处理:**硬伤必须清零**——补图 meta、清待补、对齐登记;**建议项**多数转给
   hydro-deai-polish(AI味)或 project-steward(复现)处理。
3. 处理后重跑,直到无硬伤(退出码 0),再交付。
4. 报告存进 `logs/review_report.md`,作为审计留痕。

## 作为后台/自动触发
可作为一个专职"审查子 agent",在成稿后自动触发(或用 Cowork 的定时任务定期扫描进行中的
项目),把上述硬伤在你还没交付前就标出来。它不改稿,只把问题推到你面前。

## 与其他技能的配合
- 内容质量审稿(前言/摘要/结论等五部分,含学术判断)→ hhu-thesis-review。
- AI味残留的实际清理 → hydro-deai-polish。
- 可追溯性/复现的底座 → hydro-project-steward(图 meta、登记、freeze 由它维护)。
