# 水利/水电科研工作台 · 总协调

本技能是这套 skill 的"调度中枢"。面对一个跨环节的大任务时,先用它规划流程、定默认,
再把每一步交给套件里的专项 skill 去做。它自己不产出正文、图、Word,只统筹。

## 默认约定(以中文为主)

除非用户明确另有要求,一律按下列默认执行:

1. **语言与体裁**:默认中文。默认目标是中文学位论文 / 中文核心期刊 / 中文基金申请书。
   作者若在河海大学,学位论文成文与审稿走河海专用规格。
2. **制图**:默认中文期刊惯例——`apply_style(frame="box")`(四边框)、中文宋体 +
   英文/数字 Times New Roman、变量斜体、双语题注、章.序编号。**只有**用户明确说投英文
   SCI / 国际期刊时,才切到 `frame="minimal"` 并相应调整。
3. **引用与格式**:参考文献按 GB/T 7714—2015 顺序编码制,上标数字引用。
4. **不编造(全局铁律)**:任何数据、文献、图、结论,凡作者未提供、又推不出来的,一律标
   `【待补:...】`,汇总成文末"待补清单"交回作者。任何环节都不得虚构。**且绝不提供或建议
   Sci-Hub 等盗版/绕过版权的全文获取路径**——全文只走合法 OA(Unpaywall)或学校授权渠道。
5. **成文前先定体裁与格式**:动笔写任何成品(综述/章节/摘要/申请书)前,必须先明确
   目标体裁与最终交付格式(Markdown 初稿?Word 成品?河海格式?)。用户未说明就先问一句,
   **即便用户给了"做这几步"的详细指令,也不要跳过这个确认**。只输出 Markdown 初稿时,
   要显式说明"这是初稿、如何进一步成文(如交 hhu-thesis-docx 出河海 Word)"。

若这些默认与用户需求不符(如"我要投英文期刊""这不是河海的"),以用户为准并说明将如何调整。

## 端到端流程与分派

**贯穿全程的底座:hydro-project-steward(可复现工作区)。** 任何会产出数据/图/稿件的多步
课题,开工先用它建标准项目目录;之后每步的产出都经它登记归档、记日志;成文前跑复现自检。
它让"可复现、可审计"成为机制。下列各步的产出,都应登记进项目工作区。

根据任务落在流程的哪一段,调用对应专项 skill(它们会各自自动触发,这里给出推荐顺序):

1. **读文献** → `hydro-pdf-explore`:勘察并精读文献 PDF,抽取结论与数据。
2. **综述** → `hydro-literature-review`:分主题、述评结合,凝练研究缺口。
3. **谋篇** → `hydro-paper-narrative`:定主线、摘要范式、各章承接与结论呼应。
4. **数据分析** → `hydro-data-analysis`:把原始实测/模拟数据算成特征参量、拟合与优度
   (级配、冲刷发展、误差指标、调度指标等),供制图与写作使用。
5. **写作** → `hydro-academic-writing`:按体裁范式与文风写初稿(区分实验/建模范式、
   水利/水电方向)。
6. **制图** → `hydro-figure-composer`(依 `hydro-figure-style` 规范):把真实数据画成
   规范插图,导出 SVG/PDF/PNG。
7. **审稿自查** → `hhu-thesis-review`(河海博士论文):对前言/摘要/问题提出/结论/参考文献
   五部分分级审稿并出 Word。
8. **自动审查** → `hydro-auto-review`(成稿后):扫描项目工作区的机器可查硬伤(图缺 meta、
   待补未清、登记不一致、AI味残留、复现未锁),只标不改,硬伤清零后再交付。
9. **去 AI 味润色** → `hydro-deai-polish`(成文前必经):中性学术模式,只改文风、不改事实。
10. **成文交付** → `hhu-thesis-docx`(河海学位论文):生成符合河海大学格式的 Word 成品。

不必每次都跑全流程——按用户任务只走需要的环节。例如"帮我把这章数据画成图并写一段解读",
就只走制图 + 写作两步。

## 统筹时的做法

1. **先确认关键前提**(若用户未说明):体裁(论文/期刊/申请书)、学科侧重(水利/水电)、
   是否河海、中文还是英文目标。一次问清,不逐条追问。
2. **先给流程与骨架再展开**:对大任务,先输出"打算分几步、每步用哪个 skill、产出什么",
   与用户确认后再逐步执行;长文档分环节交付。
3. **保持素材驱动**:严格依据作者提供的数据/文献/初稿;缺口标 `【待补:...】`。
4. **交回清单**:任务结束时汇总所有待补项,让作者清楚还差什么、下一步做什么。
