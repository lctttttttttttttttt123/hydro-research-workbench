# 水利/水电文献综述

## 做什么
把某个水利/水电主题的已有研究,组织成符合本领域惯例的综述:不是文献流水账,而是**分
主题、有评述、末尾凝练缺口**,自然引出研究工作。可产出两种成品:学位论文的"国内外研究
进展"章,或申请书的"立项依据—国内外研究现状及发展动态分析"。

## 铁律:绝不编造文献
这是本技能的第一原则。任何文献(作者、题名、年份、期刊、结论)**只能来自**:作者提供
的文献、用户上传的 PDF、或可检索到并经核实的真实来源。凡无法核实者,一律标
`【待补:文献——支持该论断的代表性研究】`,由作者补齐。宁可留占位,不可造假引用。
不确定某文献是否真实存在时,当作不存在处理。

## 综述的组织范式(本领域惯例)

综述**按主题分节**,不按时间流水账。每个子主题一节,内部结构:
1. 该子主题下已有研究做了什么(代表性工作、方法路线)。
2. **评**——各路线的适用性、结论分歧与局限(这一步最关键,是"综述"区别于"罗列"处)。
3. 过渡到下一子主题。

综述**末尾必设一节**凝练现有研究的不足与本文/本项目的切入点:
- 学位论文里叫"尚存在的问题";申请书里叫"现有研究的不足与本项目的切入点"。
- 这一节把前面各主题的缺口收拢,论证"为什么要做这项研究",直接引出研究内容/关键科学问题。

### 分主题的示例(按研究对象拆)
以"结构物周围局部冲刷"为例,可拆为:床面地形冲淤变化 → 局部冲刷深度 → 周围流场特征
→ 冲刷及流场的数值模拟 → 尚存在的问题。以"梯级水电站优化调度"为例,可拆为:负荷
分配模型 → 多时间尺度调度 → 模型求解算法 → 新能源背景下的水电灵活性 → 现有研究不足。
拆分维度按"研究对象/科学问题"走,使每一节聚焦一个可评述的方向。

## 检索策略(水利/水电)

先与作者确认主题边界与时间范围,再规划检索。

**若已连接 hydro-litsearch MCP 连接器**(推荐):优先用它自动检索公开题录并核实引用——
`search_papers` 按主题检索(OpenAlex,含中英文水利文献)、`get_citations` 顺引文找相关、
`verify_reference` 用 Crossref 核实 DOI、`find_open_fulltext` 用 Unpaywall 找合法免费全文。
它只取公开元数据、不下载全文;最终产出含 DOI 的清单,由作者手动去授权渠道下载。
对知网/百度学术/WoS 等无开放 API 的库,用连接器的 `build_manual_query` 生成检索式,由
作者本人手动检索回填——绝不自动抓取订阅库。

**若未连接该连接器**:基于作者提供的文献与上传 PDF 来综述,并标注还需补检的方向;
本技能不臆造检索结果。

- 中文源:中国知网(CNKI)、万方、维普;关键期刊如《水利学报》《泥沙研究》《水力发电
  学报》《水科学进展》《河海大学学报》等(具体以主题相关刊为准)。
- 英文源:Web of Science、ScienceDirect、Scopus、Google Scholar;关键期刊如
  Journal of Hydraulic Engineering、Water Resources Research、Journal of
  Hydrology、Coastal Engineering、Renewable Energy(视主题而定)。
- 检索词:中英文对照 + 同义扩展(如"局部冲刷/local scour""梯级水电站/cascade
  hydropower""负荷分配/load distribution")。
- 每条纳入的文献都要能追溯到真实出处;拿不准的标待补。

## 引用与著录
- 正文引用用上标数字、按出现顺序编号,可合并:`……普遍发生清水冲刷与河床粗化[2, 3]。`
- 参考文献按引用顺序著录,格式统一(期刊 [J]、学位论文 [D]、专著 [M]、标准 [S] 等)。
- 详细规范见 hydro-academic-writing 技能的 style-conventions。

## 工作流程
1. 与作者确认:主题、边界、时间范围、目标体裁(论文章 or 申请书综述)、已有文献。
2. 规划分主题框架(3~5 个子主题 + 末尾缺口节),先给作者确认框架再展开。
3. 逐主题综述:述 + 评,密集但真实地引用;缺口处标待补。
4. 写末尾"尚存在的问题/切入点",收拢缺口、引出研究工作。
5. 汇总参考文献(仅真实来源)与文末"待补清单"。

## 与其他技能的配合
- 读文献 PDF、抽取其结论与数据 → 用 hydro-pdf-explore 技能。
- 综述写成后并入论文/申请书的整体结构与文风 → 用 hydro-academic-writing 技能。
