# 水利/水电论文叙事逻辑

## 做什么
让一篇论文/申请书"讲得连贯、逻辑闭环":从摘要、绪论、各章、到结论,有一条清晰主线贯穿,
研究内容—结果—结论首尾呼应,而不是各章各说各话。管的是**谋篇与叙事**(怎么讲这个研究),
与管结构模板和文风的 hydro-academic-writing 互补。

## 一、一篇论文的主线该长什么样
好的水利/水电论文有一条能一句话说清的主线,并贯穿全篇。两种常见主线:
- **实验/机理类**:提出问题(某现象/关键量难以预测)→ 用实验+数值+理论去测/算 →
  逐章揭示现象、参量规律、机理 → 建立公式 → 结论回答最初的问题。
- **建模/优化类**:面向需求(如新能源消纳)提出问题 → 一条研究主线(如"精细化模拟—
  多尺度建模—求解方法—应用实例")→ 每章一个具名模块推进 → 案例验证 → 结论给出方法与
  量化改善。

判断主线是否成立的检验:摘要里的"研究方法/主线"一句,能不能和后面每一章一一对上?对不上
就说明谋篇散了。

## 二、摘要范式(叙事的浓缩)
摘要是全篇叙事的最小完整版。两种范式:
- **结论式**(实验/机理类常用):背景与问题 → 方法(采用……相结合的方法)→
  "主要研究结论如下:" → 分条,每条以动词短语领起(揭示了/建立了/探明了)+ 量化结果。
- **创新点式**(建模/优化类常用):背景与需求 → 问题 → 研究主线 → "论文的创新点主要
  体现在如下 N 个方面:" → 分条,每条一句加粗的创新点陈述领起 + 所提方法 + 量化改善。

两种都要求每条**有动词、有数据**,不写"取得较好效果"这类空话。摘要写好后,它就是检查
全篇叙事的标尺。

## 三、绪论怎么把读者带进来
绪论要完成一条说服链:这个问题为什么重要(问题的提出/研究意义)→ 别人做到哪了、还差
什么(分主题综述 + 尚存在的问题/切入点)→ 所以本文要做这几件事(研究内容)→ 怎么串起来
做(技术路线图)。研究内容分条,须与后面的结果/模块章**一一对应**——这是绪论与正文咬合的
关键。建模/优化类常在绪论加"研究区概况",介绍作为案例的真实系统。

## 四、各章之间怎么承接
- 每章开头用一两句交代:上一章解决了什么、本章要解决什么、与主线什么关系(承上启下)。
- 每章末尾"本章小结"分条量化收束,并自然过渡到下一章要处理的问题。
- 结果/模块章的顺序应体现递进(如:先地形现象 → 再局部深度 → 再流场机理 → 再公式;
  或:先精细化模拟 → 再建模 → 再求解 → 再应用),而非平行罗列。

## 五、结论怎么呼应开头
- **主要结论**分条,回答绪论提出的每一个问题;应与摘要分条呼应但更完整,不引入新数据。
- **创新点**(若单列)凝练真正的新东西(新机理/新方法/新对象),与摘要创新点一致。
- **研究展望**指出局限与后续方向,与绪论"尚存在的问题"形成首尾照应。
- 检验闭环:绪论提的问题,结论是否条条有回应?摘要的每条,正文是否都有出处支撑?

## 六、常见叙事病与修法
- **散**:各章像独立小论文,无主线——补主线句,让每章开头点明与主线的关系。
- **内容与结果对不上**:研究内容 3 条,结果却是另外几章——重排使二者一一映射。
- **结论空**:只说"研究了、效果好"——改为动词短语+量化数据的分条结论。
- **虎头蛇尾**:绪论提了大问题,结论没回答——回填结论,逐条回应绪论。
- **重复**:摘要、小结、结论大段雷同——分层:摘要最简、小结管本章、结论管全篇且更完整。

## 铁律
叙事再顺,也不能靠编造支撑。任何为"讲圆故事"而需要的数据、文献、结论,若作者未提供,
一律标 `【待补:...】`,不虚构。
