# 水利/水电 PDF 勘察与精读

## 做什么
把一篇几十上百页的水利/水电 PDF,先变成一张"结构地图",再据图定点精读,抽取真正需要的
信息(结论、关键数据、公式、图表、方法)。核心原则:**先勘察、后精读**,不盲目全文倾倒;
**只引用文中真实出现的内容**,不臆造、不脑补。

## 第一步:勘察(建地图)
对每个 PDF 先运行勘察脚本,得到基本信息、关键区块页码、章节结构、图表清单:

```bash
python3 scripts/hydro_pdf_probe.py <paper.pdf>
# 需要全文时另存:  python3 scripts/hydro_pdf_probe.py <paper.pdf> --dump-text out.txt
```

脚本会告诉你:摘要/ABSTRACT/目录/结论/参考文献等各在第几页,章节和小节标题及页码,
以及所有"图 X.X / 表 X.X"题注。据此决定接下来精读哪几页,而不是把 180 页全读进来。

若脚本报告"文字层:无",说明是扫描件,`pdftotext` 取不到字——改用下面的"看图页"方式,
必要时对页面做 OCR。

## 第二步:定点精读
根据地图,只读需要的页:

```bash
# 精读某节(如摘要在 8-9 页、结论在 141-146 页)——保留版式
pdftotext -layout -f <起页> -l <止页> <paper.pdf> -
```

**含图、公式、复杂表格的页**:文字层抽不出图形和公式,必须把页面转成图片再用视觉阅读:

```bash
pdftoppm -jpeg -r 150 -f <页> -l <页> <paper.pdf> /tmp/p
ls /tmp/p-*.jpg        # 文件名按总页数补零,别猜,ls 确认
```
然后以视觉方式阅读 /tmp/p-*.jpg,理解流场矢量图、冲淤地形图、级配曲线、调度曲线、
公式推导等文字层看不到的内容。

## 抽取要点时的规矩
- **忠于原文**:结论、数据、公式只转述文中真实出现者;数值、单位、区间照抄不改。
- **标注出处**:抽取的每条要点尽量记下所在页,便于作者复核。
- **不确定就存疑**:文字层可能有乱码(尤其中文 CJK 字体映射问题)。若某处文字明显乱码
  或存疑,转成图片用视觉核对,不要照乱码猜测。
- **不臆造**:文中没有的结论、数据、文献,一律不写;需要而缺失的标 `【待补:...】`。

## 典型场景
- **单篇消化**:勘察 → 读摘要+结论+本章小结,快速掌握全文贡献,再按需深入某章。
- **批量文献**:对一批 PDF 逐篇勘察,抽取各自的核心结论与关键数据,汇总成对比表,
  供文献综述使用(配合 hydro-literature-review 技能)。
- **规范查读**:定位到相关条文页精读,提取设计标准/计算方法——但规范编号与现行有效性
  须由作者核实,见 domain-terms 关于规范的提醒。
- **抽取数据/图表复用**:定位图表页,读出数据或说明,交给 hydro-figure-composer 重绘。

## 与其他技能的配合
- 抽取后要综述 → hydro-literature-review。
- 抽取的图要重画 → hydro-figure-composer。
- 读懂后要仿其结构文风写作 → hydro-academic-writing / hydro-paper-narrative。

## 注意
本技能只读不改。若需对 PDF 做合并、拆分、填表、加水印等操作,改用 pdf 技能。
